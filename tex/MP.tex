\begin{frame}

\frametitle{Select \& Merge p.208, ll.6-11}
派生 (derivation) の過程において, 選択 (Select) と併合 (Merge) の適用を仮定することは自然である.
\begin{itemize}
    \item 選択が適用されなければ, 数え上げ (Numeration) の指標が減らず (i.e. 使用されないLI $\in N$ : Numeration が存在する), 派生が収束しない. 当然, 経済性の議論も不要である.
    \item  併合の操作が十分に行われない場合, LFにおいて収束しない.
\end{itemize}
\begin{Rem}{予想$^\text{lx}$1 p.207.}
$\Sigma$がLFインターフェースで解釈可能 $\Rightarrow$ $\Sigma$が単一の統語対象 （Syntactic object) から成る\\  ($\Leftarrow$は成り立たない)
\end{Rem}

\end{frame}

\begin{frame}
\frametitle{Select \& Merge p.208, ll.12-17}

したがって，選択と併合は「コストゼロ」といってよい（すなわち，派生および経済性について考える余地がない）．

\[
D \coloneqq \{\, d \mid d \text{ derives from } L \,\},
\qquad
D_{C} \coloneqq \{\, d \in D \mid d \text{ converges} \,\}, 
\]
\[
D_{A} \coloneqq \{\, d \in D \mid d \text{ is admissible} \,\}.
\]

\[
D_{A} \subset D_{C} \subset D
\]

ここではp.202の議論において$L=N$として \[D\setminus (D_{C}\cup D_{A}) = D \setminus D_{C}\]に注目している.

\end{frame}


\begin{frame}
\frametitle{Numeration p.208, ll.18-30}
 \begin{itemize}
  \item 「なぜ``ある数え上げ$N$''が選ばれたのか」という問は無意味である. 心的メカニズムの解明において気にするべきではない (cf. 構造主義).
  \item $N = \{ \cdots, l_{i}, \cdots \}$, $i\ge 2$ の場合, $l$の選択において$j$回目の適用と$k$回目の適用 ($j\ne k$) が区別されるので, その結果得られる統語対象はLFにおいて異なる性
質を持つはずである.
\end{itemize}

\begin{Ex}{例2.}
$N =$ \{he$_{2}$, thinks$_{1}$, is$_{1}$, smart$_{1}$\} $\leadsto$ he$_{j}$ thinks he$_{k}$ is smart\\
(代名詞$\mathit{he}$の現れはLFにおいて区別される)
\end{Ex}

\end{frame}


\begin{frame}
\frametitle{Numeration p.208, ll.31-34}
結局, $l_{j}$と$l_{k}$が$C_{\mathrm{HL}}$  (Computational system of human language) から区別されるということに他ならない (cf. GBにおけるindex).\\ この場合, 包括性の条件 (Inclusiveness Condition) に違反するが, それでもなお自然言語の特性として重視すべきであり, BOCに還元される可能性も捨て置けない.

\begin{frame}
\frametitle{Reference set p.208, ll.35-40 \& p.209, ll.1-2}

経済性の原理を想定する限りは, 「経済性の評価において上手く順序が入る (i.e. 比較可能律を満たす) 集合」が必要になる. これを参照集合 (Reference set) と呼ぶ. \\

初期状態$A$ （数え上げ$N$など） は, 派生の入力として$\pi$（PF）と$\lambda$（LF）のインターフェースに適合するだけでなく, 参照集合も定める必要がある.  
\begin{itemize}
 \item 同じ数え上げ$N$を持つ派生のみを比較対象としなければならない. 実際 \\
N = \{ John$_{1}$, left$_{1}$\}, N' = \{ Mary$_{1}$, said$_{1}$, John$_{1}$, left$_{1}$\} を入力とする派生
\begin{enumerate}
 \item[(0)] John left. 
 \item[(1)] Mary said John left.
\end{enumerate}
は, (1)のほうが(0)に比べ併合の適用回数が多く, 明らかに経済的でない.
\end{itemize}

\end{frame}


\begin{frame}
\frametitle{Reference set p.209, ll.3-14}

\begin{itemize}
 \item しかし入力を同じ数え上げに制限したところで, 全派生の比較は計算量が膨大になり, 現実的でない.\\
\quad そこで局所的な解釈を採用する. すなわち, 派生のある段階$\Sigma$において, そこからの操作（残りの数え上げに基づく部分派生）のみを比較対象とし, ある操作OPについてより経済的な派生が存在する場合, OPの適用を禁止する. 派生が進むにつれ参照集合は小さくなる.
\end{itemize}
 
\end{frame}

\begin{frame}
\frametitle{Inclusiveness condition p.209, ll.39-40}

\begin{itemize}
 \item $N$からPFへの計算$\colon N \to \pi$ は包括性の条件を満たさない. 
\end{itemize}
\begin{marker}
\textbf{Note 10.}音節やイントネーションの情報はLIに含まれず, 通常A-Pにおいては, 語彙入力と音声出力が大きく異なる.\\
Halle $\&$ Marants の分散形態論 (Distributed morphology) では, 音韻素性が$N \to \lambda$の計算に現れないと想定する. この場合, \\``犬''と``猫''はPFでのみ区別される.
\end{marker}
\begin{itemize}
 \item 単一のLIを複数回選択する場合, LFでは何らかの区別が必要である. \\
(BOCへの還元可能性 $\leadsto$ ``小さな逸脱'' (narrow departure))
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Inclusiveness condition p.210, ll.1-5}

\begin{itemize}
 \item ある対象を``インターフェースにおいて不可視''にする操作として削除 (Delete $\alpha$) を想定する. これも包括性原理に違反するが, 「あるインターフェースで削除されてもなお$C_{\mathrm{HL}}$はアクセス可能である」と仮定しておく. (67(108))
\end{itemize}

\begin{enumerate}
 \item[(a)] [which picture of himself] did John say [$t'$ that [Bill liked t best]]
 \item[(b)] [which pictures of each other] did they say [$t'$ that [we liked t best]]
\end{enumerate}

\begin{marker}
\textbf{Note 11.}(a)において$\textit{John}$, $\textit{Bill}$の両方が$\textit{himself}$の先行詞に成りうる. (b)も$\textit{they}$, $\textit{we}$と$\textit{each other}$について同様.\\
したがって束縛原理Aより, 中間トレース$t'$もLFにおいて可視でなければならない (cf. 連鎖束縛).\\(i.e. PFでは不可視だがLFでは可視である)
\end{marker}

\end{frame}

\begin{frame}
\frametitle{Feature checking p.210, ll.6-11}

$C_{\mathrm{HL}}$の中核をなす操作として$\textbf{素性照合}$($\textbf{Feature checking}$)[最後の手段 (Last resort) の条件の下で移動を引き起こす操作]を想定する.\\
素性照合を理解するため, まず手始めに削除に還元する方針をとる. すなわち照合された素性は``インターフェースにおいて不可視''になると考える.


\begin{marker}
\textbf{Note 12.}\;ある統語対象$\alpha$とその痕跡 (コピー) $t_{\alpha}$について, $\alpha$が素性照合された後の同一性を担保するために, 連鎖 (chain) のある要素が素性照合された場合, すべての要素が素性照合されると考える.
\end{marker}

\qquad
\begin{enumerate}
\item [(c)] we are likely [$t_{3}$ to be asked [$t_{2}$ to [$t_{1}$ build airplanes]]] (260(54))
\end{enumerate}


\end{frame}


\begin{frame}
\frametitle{Bare output conditions p.210, ll.12-22}
簡単のためある数え上げ$N$を固定する. このとき$N \to \pi, N \to \lambda$について
\[
\{\, \pi \mid \pi \text{ is interpretable at PF} \,\}
\;\neq\;
\{\, \lambda \mid \lambda \text{ is interpretable at LF} \,\}
\]

$\leadsto$ $C_{\mathrm{HL}}$による言語計算はある時点で``枝分かれ''する.\\

\begin{enumerate}

\item[(1)] 一度枝分かれしたら, それ以上は互いに影響しない.
\item[(2)] 全体を通じて一様である. すなわち, 任意の操作が任意の段階で適用されうる.

\end{enumerate}
(1)は採用するが(2)は$N \to \lambda$における計算にのみ想定する. 
\begin{itemize}
\item $N \to \pi$における計算はLIの内部構造 (音韻素性など) も含め修正しうる.
\item A-PとC-Iの要請する出力条件の違いから, 計算の非対称性は明らかである.
\end{itemize}


\end{frame}


\begin{frame}
\frametitle{Spell-Out p.210, ll.23-30}

$N \to \lambda$の計算のために, A-Pインターフェースでのみ必要となる要素を言語計算のある段階で取り除く操作として\textbf{書き出し (Spell-Out)}を考える.

%%Spell-Outの概要
\[
C_{\mathrm{HL}}:
N  \stackrel{{\color{blue}\text{(O)}}}{\rightarrow} \Sigma 
\stackrel{\text{Spell-Out}}{\longrightarrow}
\begin{cases}
\Sigma_{\mathrm{L}} \stackrel{{\color{blue}\text{(C)}}}{\rightarrow} \lambda  \\[4pt]
\Sigma \stackrel{{\color{blue}\text{(P)}}}{\rightarrow} \pi 
\end{cases}
\]

\begin{itemize}
\item $C_{\mathrm{HL}}$のサブシステムとして, 言語計算の

\begin{enumerate}

\item[(P)] $\Sigma \to \pi$は`'音韻部門''(phonological component)
\item[(C)] $\Sigma_{\mathrm{L}} \to \lambda$は``非顕在部門''(covert)
\item[(O)] 書き出しが適用される前の$N\to \Sigma$は``顕在部門''(overt) 

\end{enumerate}
が担当すると考える.
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Spell-Out p.210, ll.31-36}
\begin{itemize}
\item ここではさらに, 書き出しを適用された$\Sigma$は形態部門に送られて, 語のような単位が構成され, 不要な素性が取り除かれたのち, 音韻的な操作によって$\pi$に収束するモデルを想定している.

\end{itemize}

\end{frame}


\begin{frame}
\frametitle{Appendix-1 : Chomsky as a logician}

\cite{Chomsky1995}の随所からChomskyの数理論理学・計算機科学への意識がうかがえる. \\
\begin{itemize}
\item 派生とは\textbf{記号列} $S = (\sigma_1,\sigma_2,\dots,\sigma_n)$ を生成する手続きのこと.
\end{itemize}
\vspace{5pt}
数理論理学では, システムに対しある条件を満たす「論理式の有限列」を形式的証明と定義する. たとえば命題論理におけるŁukasiewiczのシステムは公理系:

\begin{align*}
&\text{[A1]} \quad \varphi \to (\psi \to \varphi)\\
&\text{[A2]} \quad (\varphi \to \psi) \to [(\varphi \to (\psi \to \chi)) \to (\varphi \to \chi)]\\
&\text{[A3]} \quad (\lnot\psi \to \lnot\varphi) \to (\varphi \to \psi)\\
\end{align*}

を設定し, これを出発論理式として
\end{frame}


\begin{frame}
\frametitle{Appendix-2 : Chomsky as a logician}
推論規則Modus Ponens:
\begin{align*}
\text{MP} \quad \frac{\varphi \qquad \varphi \to \psi}{\psi} 
\end{align*}
によって新たな論理式を推論する. これは意味から切り離された文字を機械的なルールで処理するシステムであり, 論理式とはいわばマークアップ言語によって書かれたテキストデータの一種である. このように, ある形式の集合を定義することで正しさを説明するアプローチを\textbf{統語論 (Syntax)} と呼ぶ.\\
\vspace{5pt}
\quad 一方, 妥当性, すなわち「前提がすべて真ならばその結論も真となる」議論に正しさを求めるアプローチを\textbf{意味論 (Semantics)}と呼ぶ. 
\end{frame} 

\begin{frame}
\frametitle{Appendix-3 : Chomsky as a logician}
命題論理 (Łukasiewiczのシステム) において「証明可能性」と「妥当性」は一致する:

\begin{thm}{定理3 完全性定理.}
Łukasiewiczのシステムにおいて
\[
\vdash A \quad \Longleftrightarrow \quad \models A
\]
(左辺は統語論, 右辺は意味論によるアプローチ)
\end{thm}

Chomskyはこの議論を踏まえて統語操作を想定しつつ
\begin{itemize}
\item 言語計算はBOCにより制限される. すなわち, 認知システムは外側の運用システムに動機付けられる.
\end{itemize}
として自然言語と形式言語を明確に区別する.
\end{frame}


\begin{frame}
\frametitle{Appendix-4 : Chomsky as a logician}
Chomskyは数理論理学・計算機科学へ多大な影響を与えた：

\begin{itemize}
\item \textbf{Chomsky階層}:
\[
\text{正規} \subset \text{文脈自由} \subset \text{文脈依存} \subset \text{Type-0 (RE)}
\]
\item \textbf{弱生成力}(文法が生成しうる文字列集合)と\textbf{強生成力}(文に対応づけられる構造)
\item \textbf{Chomsky標準形 (Chomsky normal form)}
\item 文脈自由文法を強い意味で等価な非決定性プッシュダウン・オートマトンへ写像するための構成的手続きの存在 (Langendoenの学部卒業論文)
\end{itemize}

これは数理論理学・計算機科学の一分野である形式言語理論の基礎となり,  
この分野のすべての教科書にChomskyの名が登場すると言っても過言ではない.

\end{frame}

\begin{frame}
\frametitle{Appendix-5 : Chomsky as a logician}
数学に基礎を求める厳密科学としての言語学といえば

\begin{itemize}
\item Montagueに始まる形式意味論
\item Steedmanによる組み合わせ範疇文法 (Combinatory Categorial Grammar, CCG) 
\end{itemize}

などが代表的とされるが, ここにChomskyの数学者・論理学者としての貢献も凄まじいことを強調しておく.\\
\vspace{5pt}
近年Chomskyが数理論理学的なアプローチに興味を失っていること\cite{Chomsky2003}は残念だが, これからも数学との相互作用に期待したい.

\begin{frame}

\begin{Def}{定義1 句構造文法.}
次のシステム$G\coloneq\langle V_\text{N}, V_\text{T}, S, P\rangle$を\textbf{句構造文法}という.
\begin{itemize}
\item $V_\text{N}$はアルファベット, その元を非終端記号 (nonterminal symbol) または変数という.
\item $V_\text{T}$もアルファベットで $V_\text{N}\cap V_\text{T}=\emptyset$を満たす. またその元を終端記号 (terminal symbol) という.
\item $S\in V_\text{N}$で, 開始記号という.
\item $P$は$u\to v$という形の式の集合である. ただし$u,v\in (V_\text{N}\cup V_\text{T})^*$ (すなわち$V_\text{N}\cup V_\text{T}$上の語全体の集合) で, かつ$u$は少なくとも一つ変数を含む. $P$の元を生成規則という.
\end{itemize}
\end{Def}
\end{frame}

\begin{frame}
\frametitle{The development of X-bar theory p.222}
句構造文法は単に文法ともよぶ. さてこれで文脈自由文法が定義できる.

\begin{Def}{定義2 文脈自由文法.}
文法$G=\langle V_\text{N}, V_\text{T}, S, P\rangle$, $V\coloneq V_\text{N}\cup V_\text{T}$において, $P$の生成規則が
\[
A\to \beta \hspace{20pt}(A\in V_\text{N}, \beta\in V^*)
\]
という形の物だけであるとき\footnote{正確には, 開始記号$S$以外の右辺には空語$\lambda$が現れないという制約をつける.}, $G$は\textbf{文脈自由文法 (Context-Free Grammar; CFG)}であるといい, $G$によって生成される言語を\textbf{文脈自由言語(Context-Free Language; CFL)}という.
\end{Def}
\end{frame}

\[
\langle \alpha, \beta \rangle \quad (\alpha = \beta)
\]
となり, 連鎖の二要素が同一になってしまう.

\begin{marker}
数学において$\langle \cdot, \cdot \rangle$は順序対(ordered pair)を表す(i.e. $\langle \alpha, \beta \rangle \neq \langle \beta, \alpha \rangle$). 普通, $\alpha$と$\beta$が異なることに意味のある文脈で用いられるため, 上の表記は数学的には不自然であり, 生成文法特有の表記であることに注意したい.

\medskip
ちなみに順序対の集合論的定義としては, Kuratowskiによる
\[
\langle \alpha, \beta \rangle \coloneq \{\{\alpha\}, \{\alpha, \beta\}\}
\]
というものが一般的である.
\end{marker}

\end{frame}

\Sigma \mapsto \Sigma'\; ; \Sigma\:\text{内の}M\:\text{を標的にして}\:\alpha\:\text{を繰り上げる}
\]

$\Sigma'$で$M$は$\left\{N, \left\{\alpha, M\right\}\right\}$に置き換えられたとする. $\Sigma'$には同一の$\alpha$が2つ含まれるが

\[
(\Sigma\text{における}\alpha\text{のco-constituent})
\leftrightarrow t(\alpha),\:M \leftrightarrow \alpha
\]

という対応によって痕跡理論と同様に区別できる.
\end{Ex}
\end{frame}


\begin{frame}

(14)$\to$(15)での移動によって現れる二つの $\alpha$は, 形式的にはそれぞれ次のように区別される$\colon$

\medskip
\begin{itemize}
\item 元の位置：
\[
\langle \alpha, \beta \rangle
\]
（$\beta$ は移動前における $\alpha$ のco-constituent）

\item 移動先の位置：
\[
\langle \alpha, K \rangle
\]
（$K$ は移動後における $\alpha$ のco-constituent, $K=\{\beta, \{\beta, \alpha\}\}$）
\end{itemize}

\vspace{1em}

$\alpha$ 自体は同一でも位置関係が異なるため，二つの項は区別される．

\end{frame}

\begin{frame}
繰り返しになるが, ここまでの議論をまとめておこう.

\begin{Def}{定義$^\text{lx}5$ 移動する要素の位置.}
$\Sigma, \Sigma'\colon$派生の段階\\
$\alpha\colon\Sigma$の項, $\beta\colon\alpha$の($\Sigma$における) co-constituentである項\\
$K\colon\Sigma$に含まれる統語対象

\vspace{0.5em}

$K$を標的にして$\alpha$を繰り上げる移動$\Sigma\mapsto \Sigma'$について, $\alpha$の移動前の位置$\mathrm{POS}_1$, 移動後の位置$\mathrm{POS}_2$を次のように定義する$\colon$

\[
\left\{
\begin{aligned}
\mathrm{POS}_1 &\coloneq \left\langle \alpha, \beta \right\rangle \\
\mathrm{POS}_2 &\coloneq \left\langle \alpha, K \right\rangle 
\end{aligned}
\right.
\]

\end{Def}
\end{frame}


\begin{frame}


\begin{Def}{定義$^\text{lx}6$ 連鎖$\mathrm{CH}$.}
連鎖$\mathrm{CH}$は同一要素$\alpha$が現れる二つの位置の組として次のように定義される$\colon$

\[
\mathrm{CH} \coloneq \langle \text{POS}_1, \text{POS}_2 \rangle\: \left(= \left\langle \langle \alpha, \beta \rangle,  \langle \alpha, K \rangle\right\rangle\right)
\]
\end{Def}

\medskip
\begin{itemize}
\item より簡潔に, 連鎖を $(K, \beta)$ と考えてもよい.
\item 慣例的には, この連鎖を$(\alpha, t(\alpha))$と表記する．
\end{itemize}

\begin{marker}
\begin{itemize}
\item 「痕跡を新たに導入する」という意味ではなく、単に記法の問題である．
\item 数学において$(\cdot, \cdot)$も順序対であり, 例えば数ベクトルを表すために用いられる.
\end{itemize}
\end{marker}

\end{frame}
